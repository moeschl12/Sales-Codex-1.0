# Rule: Repository Authority

The Sales Codex repository is the only source of truth.

Required behavior:

- Use existing files before making claims.
- Do not invent sources, facts, object IDs, or repository state.
- Mark uncertainty instead of smoothing it away.
- Document contradictions instead of resolving them without authorization.
- Treat chat context as temporary and non-authoritative.

Primary references:

- `AGENTS.md`
- `PROJECT_BOOTSTRAP.md`
- `CURRENT_STATE.md`
- `SESSION_BRIEF.md`
- `00_project/NEXT_ACTION.md`
