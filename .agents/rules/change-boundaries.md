# Rule: Change Boundaries

Default boundary: change the smallest set of files needed for the explicit request.

Never do these without explicit editor authorization:

- Change framework methodology.
- Add new templates.
- Reclassify evidence levels as a side effect.
- Create new knowledge object categories.
- Close open decisions.
- Start a new macroproject or research project.
- Modify `03_knowledge_base/` outside the active task scope.

For structure or architecture questions, prefer a proposal unless Felix explicitly asks to implement.
