# Rule: Knowledge Work

Knowledge work follows the Sales Codex pipeline:

SRC -> ST -> A -> MEC -> P -> T -> MOD -> VAL -> Abschlussbericht

Core constraints:

- ST objects come from primary text, not memory.
- A objects can relate to multiple ST objects.
- MEC objects require canonicalization against existing mechanisms.
- P objects must not be reformulations of a single mechanism.
- T objects must describe concrete repeatable action, not broad competence.
- VAL review is part of the workflow, not optional decoration.
