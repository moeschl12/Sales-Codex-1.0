# Rule: Session Start

Before writing project files, load the minimum required context.

Standard sequence:

1. Read `PROJECT_BOOTSTRAP.md`.
2. Read `SESSION_BRIEF.md` if present.
3. Read `00_project/NEXT_ACTION.md`.
4. Identify the likely task type from `TASK_TYPES.md`.
5. Read only the directly relevant task files.

Do not run broad scans over `00_project/` or `03_knowledge_base/` unless the active task type or explicit editor instruction allows it.

If the current task is an ad hoc editor instruction outside the closed V1.1 launcher, keep the action scoped and do not infer a new macroproject.
