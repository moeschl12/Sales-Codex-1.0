# Command Recipe: Review

Use for code, repository, object, governance, or consistency review requests.

Default stance:

- Findings first.
- Order by severity.
- Cite concrete files and lines where possible.
- Separate confirmed issues from open questions.
- Do not rewrite content unless the user asks for implementation.

Sales Codex mapping:

- `T5_REVIEW` for validation-style reviews that produce a VAL report.
- `T6_AUDIT` for broader read-only audits with explicit scope.
- `T7_KONSISTENZ` for one rule across one object type.
