# Command Recipe: Architecture Proposal

Use for structure, workflow, control-plane, or refactoring questions.

Default stance:

1. Inspect structure metadata and directly relevant governance files.
2. Identify whether the request changes methodology or only adds a helper layer.
3. If methodology changes, produce a proposal and wait for editor approval.
4. If Felix explicitly asks to implement a helper layer, keep it subordinate and documented.

Output:

- Current state.
- Proposed change.
- Risks and boundaries.
- Files changed or proposed.

Task type:

- Usually `T4_ARCHITEKTUR`.
