# V11-06 — Research Portfolio Wave 2

Status: LATER  
Owner: Executor Agent  
Reviewer: Independent Scientific Auditor  
Editor: Felix

---

## 1. Mission

Execute a controlled second research wave on the highest-leverage validated gaps, using full Research Lifecycle discipline, falsification protocol and context resets between W-projects.

---

## 2. Scope

Allowed work:

- Re-read current Research Portfolio and Scientific Debt.
- Select research candidates based on V11-02 and V11-05.
- Propose the wave composition for Editor confirmation if new W-project activation is required.
- Execute approved W-projects through the existing Research Lifecycle.
- Use context reset / repository reload between W-projects.
- Conduct cross-project synthesis after the wave.

---

## 3. Non-Scope

Do not:

- start more than the approved wave scope,
- generate new theoretical models without repository impact analysis,
- treat confirmation as sufficient evidence,
- bypass Editor Decisions,
- execute multiple W-projects in one unbroken context trajectory.

---

## 4. Dependencies

- V11-02 complete.
- V11-05 complete or explicitly bypassed by Editor Decision.
- Editor approval of selected W-projects.

---

## 5. Execution Mode

Large macro-project on management level. Sequential research execution on agent level.

Internal shape:

1. Portfolio state check.
2. Candidate selection.
3. Editor confirmation if new W-projects are activated.
4. W-project lifecycle.
5. Integration and health check.
6. Context reset.
7. Next W-project if approved.
8. Cross-wave synthesis.

---

## 6. Internal Checkpoints

- Scope Check before each W-project.
- Falsification Check in each hypothesis/review phase.
- State Check before integration.
- Health Check after integration.
- Context Reset after each W-project.
- Pre-Completion Check for wave-level DoD.

---

## 7. Definition of Done

V11-06 is DONE only when:

1. Wave scope is explicitly authorized.
2. Every included W-project has full lifecycle artifacts.
3. Each Editor Decision is explicit.
4. Repository impact analysis exists for each integration.
5. Health checks pass or failures are classified.
6. Cross-wave synthesis identifies convergence, contradictions, debt and next priorities.
7. Completion report and audit package exist.

---

## 8. Audit Requirements

Audit must verify scientific falsification effort, source grounding, transfer limits, and whether the wave produced structural value rather than research momentum.
