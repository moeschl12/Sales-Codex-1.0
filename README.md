# Sales Codex Workspace

Version: 0.2  
Erstellt: 2026-06-27

## Zweck

Dieses Repository ist die zentrale Arbeitsumgebung für den Sales Codex.

Der Sales Codex ist kein normales Buchprojekt und keine Sammlung von Vertriebstipps. Ziel ist der Aufbau eines strukturierten, evidenzbewussten Wissenssystems über Vertrieb, Verkauf, Verhandlung, Käuferpsychologie, Kommunikation und kommerzielle Entscheidungsprozesse.

## Grundregel

> Das Repository ist die Quelle der Wahrheit. KI-Systeme sind Mitarbeiter, nicht Speicherorte.

Claude, ChatGPT, Gemini, Perplexity oder spätere Modelle dürfen beitragen. Dauerhaft gültiges Wissen muss jedoch immer in diesem Workspace gespeichert werden.

## Struktur

- `00_project/` — Projektsteuerung, Roadmap, Entscheidungen, Changelog
- `01_framework/` — Methodik, Standards, Wissensmodell, Agentenregeln, Templates
- `02_sources/` — Bücher, Studien, Artikel, Interviews, Podcasts und sonstige Quellen
- `03_knowledge_base/` — Aussagen, Beobachtungen, Annahmen, Prinzipien, Mechanismen, Techniken, Kompetenzen, Modelle, Meta-Modelle
- `04_book_analysis/` — strukturierte Analysen einzelner Bücher
- `05_publications/` — spätere Ausgaben: Buch, Arbeitsbuch, Trainings, Vorträge
- `06_media/` — Bilder, Diagramme, Mindmaps
- `07_scripts/` — spätere Automatisierungen
- `99_archive/` — archivierte oder verworfene Inhalte

## Arbeitsprinzip

Nicht zuerst das Buch schreiben.  
Zuerst das Wissenssystem bauen.  
Das Buch ist nur eine spätere Ausgabe der Wissensbasis.
