# Codex Knowledge Model

## Zweck

Das Wissensmodell definiert, welche Wissensobjekte im Sales Codex existieren und wie sie miteinander verbunden sind.

## Grundregel

Der Sales Codex besteht nicht primär aus Kapiteln.  
Er besteht aus Wissensobjekten.  
Kapitel sind spätere Darstellungsformen.

## Wissensobjekte

### Source

Eine Quelle: Buch, Studie, Artikel, Interview, Podcast, Datensatz oder Fallanalyse.

### Statement

Eine konkrete Aussage innerhalb einer Quelle.

### Observation

Eine beobachtete Tatsache oder Beschreibung ohne kausale Interpretation.

### Assumption

Eine Annahme, auf der ein Modell oder eine Methode beruht.

### Principle

Eine abstrahierte, wiederverwendbare Erkenntnis über Verkauf, Käuferverhalten oder Kommunikation.

### Mechanism

Die kausale Erklärung, warum ein Prinzip funktioniert.

### Technique

Ein praktisches Werkzeug zur Anwendung eines Prinzips.

### Competency

Eine trainierbare Fähigkeit, die die Anwendung von Techniken und Prinzipien ermöglicht.

### Model

Eine strukturierte Kombination aus Annahmen, Prinzipien, Techniken und Mechanismen.

### Meta-Model

Ein übergeordnetes System, das mehrere Modelle verbindet.

## Objektbeziehungen

Eine Quelle enthält Aussagen.  
Aussagen können Beobachtungen, Annahmen oder Hypothesen enthalten.  
Aus diesen werden Prinzipien extrahiert.  
Prinzipien beruhen auf Mechanismen.  
Techniken wenden Prinzipien praktisch an.  
Kompetenzen ermöglichen Techniken.  
Modelle ordnen mehrere Prinzipien und Techniken.  
Meta-Modelle verbinden mehrere Modelle.

## Wichtig

Das Buch ist nicht das Wissenssystem.  
Das Buch ist eine Ausgabe des Wissenssystems.
